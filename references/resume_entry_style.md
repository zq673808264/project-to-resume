# Resume Entry Style

## Bullet Formula

Use this shape by default:

`Verb + technical object + method/tooling + result or reason`

Examples:

- Built a course-selection workflow with ASP.NET Web Forms and SQL Server, covering student selection, score query, and administrator review flows.
- Refactored repeated database access into reusable helpers, reducing duplicate query code and making page-level logic easier to maintain.
- Added validation and role-aware navigation so student, teacher, and administrator users reach only the functions relevant to their permissions.

## Strong Verbs

Use precise verbs supported by evidence:

- Built
- Implemented
- Designed
- Integrated
- Refactored
- Automated
- Tested
- Optimized
- Migrated
- Debugged
- Documented

Avoid vague verbs:

- Participated in
- Responsible for
- Helped with
- Learned

## Chinese Bullet Patterns

Use compact Chinese resume language:

- 基于 `<tech>` 设计并实现 `<feature>`，支持 `<workflow>`，提升 `<quality/capability>`。
- 封装 `<module/helper>`，统一 `<repeated behavior>`，降低 `<maintenance cost/duplication>`。
- 接入 `<api/tool/service>`，实现 `<capability>`，完善 `<user-facing workflow>`。
- 通过 `<tests/logging/validation>` 保障 `<reliability/security/data correctness>`。
- 构建基于 `<project context>` 的 `<automation/tool>`，从 `<evidence sources>` 中提取 `<technical signals>`，生成 `<career artifact>`。

## Role-Targeted Angles

- Frontend: interaction workflow, state management, responsive layout, component reuse, accessibility, visual quality.
- Backend: API design, data model, authentication, permissions, reliability, logging, performance, integration.
- Full stack: end-to-end workflow, frontend/backend coordination, database design, deployment, testability.
- AI engineering: prompt design, model/API integration, context retrieval, tool calling, evaluation, guardrails.
- Data: data cleaning, feature extraction, modeling, evaluation, visualization, reproducibility.

## Career Stage Angles

- Campus: course/project context, independent implementation, technical breadth, learning depth, demonstrable functionality.
- Internship: practical workflow, collaboration readiness, maintainability, validation, ability to ship useful features.
- Experienced: ownership, architecture decisions, measurable reliability, scalability, cross-functional impact, risk control.

## Truthfulness Checks

Before finalizing, verify each bullet:

- Does the repository contain evidence for the technology named?
- Does the project actually include the feature?
- Is the outcome stated without unsupported numbers?
- Would the user be able to explain the claim in an interview?

## Open Question Format

When a useful claim needs confirmation, write:

`待确认：是否有可量化指标，如加载速度、用户数量、部署地址、测试覆盖率或业务使用情况？`

## AI-Assisted Project Wording

Be honest without weakening the project:

- Good: 使用 Codex/Claude Code 辅助生成初版代码，并负责需求拆解、代码审查、调试验证、模块集成与结果评估。
- Good: 构建 AI 辅助开发流程，将需求说明、代码实现、测试反馈与简历素材沉淀串联为可复用工作流。
- Avoid: 完全独立手写全部代码, unless the user confirms it.
- Avoid: AI 自动完成项目, because it erases the user's engineering work and is usually too vague.
